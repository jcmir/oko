# Roadmap разработки (Дорожная карта)

Здесь описаны этапы разработки для обеспечения модульности и отказоустойчивости системы.

## Этап 1: Фундамент и Тесты (Текущий)
- [x] Настройка кросс-компиляции (Windows + Linux).
- [x] Реализация и тестирование локальной БД (`rusqlite`, сохранение PIN-кода, оффлайн правила).
- [x] Создание структуры `NetworkManager` (базовые статусы Green, Yellow, Red).
- [x] Трансляция статусов через Named Pipe.
- [ ] Рефакторинг `core-service`: вынесение логики IPC и процессов в отдельные модули (`ipc.rs`, `process.rs`).
- [ ] Написание Unit-тестов для `core-service` и `watchdog-utility`.

## Этап 2: Трей-Агент и UI
- [ ] Доработка `tray-agent`: чтение статусов сети из Named Pipe (Парсинг строки `HEARTBEAT ... STATUS:X`).
- [ ] Динамическая смена иконок (Зеленая, Желтая, Красная) в системном трее Windows.
- [ ] Окно ввода PIN-кода для перехода в режим администратора.

## Этап 3: Сетевое взаимодействие (Связь с Go)
- [ ] Интеграция HTTP/WebSocket клиента (например, `reqwest` или `tokio-tungstenite`) в `network.rs`.
- [ ] Первичная регистрация: Генерация Machine ID и формирование QR-кода.
- [ ] Синхронизация: Получение списка блокировок и правил от Go-сервера.

## Этап 4: Anti-Kill и Геймификация
- [ ] Взаимный мониторинг: Ядро проверяет Демона, Демон проверяет Ядро.
- [ ] Блокировка диспетчера задач (добавление его в черный список при активном контроле).
- [ ] Экономика времени (Геймификация): Локальный счетчик заработанного времени, который тикает вниз при активных играх.
