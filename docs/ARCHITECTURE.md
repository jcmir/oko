# Архитектура Клиента (Rust)

Система родительского контроля для ПК состоит из нескольких независимых, но тесно связанных компонентов. Основной принцип — модульность и отказоустойчивость.

## 1. Компоненты системы

### 1.1 Core Service (`core-service`)
Ядро системы. Работает как служба Windows в контексте `SYSTEM`.
* **db.rs (База данных):** Локальная зашифрованная БД (SQLite), хранит настройки, списки заблокированных процессов и PIN-код. Позволяет системе работать в оффлайн-режиме (Yellow State).
* **network.rs (Сетевой менеджер):** Отвечает за связь с Go-сервером (WebSocket/HTTP). Отслеживает состояние сети:
  * `Green`: Подключен к серверу.
  * `Yellow`: Оффлайн, но локальные правила применяются.
  * `Red`: Ошибка соединения/демона.
* **ipc.rs (Межпроцессное взаимодействие):** Поднимает Named Pipes (именованные каналы) с защитой DACL для общения с Трей-Агентом и Демоном (Watchdog).
* **process.rs (Мониторинг):** Опрашивает процессы Windows и убивает те, которые находятся в черном списке БД.

### 1.2 Watchdog Utility (`watchdog-utility`)
Служба-демон, работающая от `SYSTEM`.
* **Цель:** Защита `core-service` от завершения (Anti-Kill).
* **Логика:** Читает "пульс" (`HEARTBEAT`) из Named Pipe ядра каждые 1-2 секунды. Если пульс пропадает или соединение рвется, Watchdog принудительно перезапускает Core Service через Windows SCM (Service Control Manager).

### 1.3 Tray Agent (`tray-agent`)
Пользовательский интерфейс (UI) для родителя и ребенка.
* Работает в сессии пользователя (User mode).
* Подключается к Named Pipe от `core-service`.
* Читает статус (`STATUS:GREEN` / `YELLOW`) и меняет цвет иконки в трее.
* Предоставляет окно для ввода PIN-кода (передает его в Core Service для верификации и временной разблокировки).

## 2. Безопасность и Anti-Tamper
1. **Разделение привилегий:** Ядро и Демон работают от `SYSTEM`. Пользователь (даже администратор без специальных навыков) не может легко изменить их файлы или остановить службы (при правильной настройке ACL).
2. **Взаимный мониторинг (В планах):** Core Service также будет проверять, жив ли Watchdog, и запускать его при необходимости.
3. **Безопасные каналы (DACL):** Named Pipes разрешают подключение только для `SYSTEM` и Администраторов (на чтение/запись).

## 3. Оффлайн-устойчивость
Если пропадает интернет:
1. `network.rs` переводит состояние в `Yellow`.
2. Ядро начинает опираться исключительно на правила из локальной БД (лимиты времени, список блокировок).
3. При появлении сети ядро синхронизирует логи и новые правила с сервером.
